# Конфигурация среды разработки V853-PRO

## 1. Установка виртуализационных инструментов

### 1.1 Установка VMware

На Windows существует множество программ виртуализации. В настоящее время на рынке популярны VMware и VirtualBox. VMware выпускается в двух версиях: платная профессиональная версия Workstation Pro и бесплатная версия Workstation Player для некоммерческого использования. Рекомендуется использовать Workstation Player.

Загрузите последнюю версию установочного пакета Workstation Player с официального сайта VMware (www.vmware.com) через браузер или используйте предоставленный пакет. Если вы уже установили VMware версии 15.x и выше, то можете пропустить шаги установки и перейти к следующему разделу для загрузки и запуска виртуальной машины.

Откройте в браузере ссылку https://www.vmware.com/products/workstation-pro/workstation-pro-evaluation.html, найдите кнопку загрузки (стрелка на скриншоте ниже), нажмите "DOWNLOAD NOW" для Windows версии VMware Workstation, чтобы начать загрузку.

### 1.2 Запуск Ubuntu с помощью VMware

Получить виртуальную машину можно двумя способами: загрузить напрямую с сайта vmwareimages или получить из нашего облачного хранилища (03_Tools/Ubuntu_18.04.6_VM.7z). Мы рекомендуем загрузить с официального сайта, чтобы ознакомиться с полным процессом.

Откройте в браузере https://www.linuxvmimages.com/images/ubuntu-1804/, найдите место, на которое указывает стрелка ниже, и нажмите "VMware Image" для загрузки.

![image-20230627110946576](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627110946576.png)

После загрузки вы получите архив Ubuntu_18.04.6_VM.7z. Распакуйте этот архив с помощью инструмента архивирования.

![image-20230627110957687](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627110957687.png)

После распаковки вы получите два файла. Файл vmx - это конфигурационный файл VMware, который нам нужен. Убедитесь, что на диске, где размещена виртуальная машина, доступно не менее 200 ГБ свободного места для проведения экспериментов.

![image-20230627111008000](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111008000.png)

Вы можете спросить, почему требуется 200 ГБ пространства. Это потому, что компиляция встроенной Linux - это сложный и требовательный процесс, который предъявляет высокие требования как к производительности компьютера, так и к операциям чтения-записи. Поэтому лучше заранее подготовить достаточно свободного места.

### 1.3 Включение виртуализации в BIOS

Большинство компьютеров уже имеют включенную виртуализацию в BIOS. Вы можете подтвердить это, открыв диспетчер задач, как показано на рисунках 3.1 и 3.2 ниже:

![image-20230627111052357](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111052357.png)

Рисунок 3.1. Открытие диспетчера задач в Windows 10

![image-20230627111107499](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111107499.png)

Рисунок 3.2. Проверка статуса виртуализации

Если виртуализация не отображается как "включена", вам необходимо перезагрузить компьютер и включить виртуализацию в BIOS. Интерфейс BIOS может отличаться в зависимости от производителя компьютера. Следующие шаги являются примером:

Шаг 1: Вход в BIOS

При включении или перезагрузке компьютера во время экрана самопроверки (POST) нажимайте клавишу F2 несколько раз (примечание: на некоторых моделях используется Fn+F2) для входа в меню настроек BIOS Setup.

Шаг 2: Найдите меню виртуализации

Используя клавиши со стрелками на клавиатуре, выберите меню "Configuration", затем используйте стрелку вниз для выбора опции "Intel Virtual Technology" и нажмите Enter, как показано на рисунке 3.3:

![image-20230627111121301](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111121301.png)

Рисунок 3.3. Поиск виртуализации в BIOS

Шаг 3: Включение виртуализации

В появившемся меню выберите "Enable" и нажмите Enter, как показано на рисунке 3.4:

![image-20230627111130934](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111130934.png)

Рисунок 3.4. Включение виртуализации

Шаг 4: Сохранение

Нажмите клавишу F10 на клавиатуре (примечание: на некоторых моделях требуется нажать Fn+F10) для вывода диалога сохранения, выберите "Yes" для сохранения и выхода, после чего компьютер автоматически перезагрузится, как показано на рисунке 3.5:

![image-20230627111142338](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111142338.png)

Рисунок 3.5. Выход из BIOS с сохранением

### 1.4 Запуск Ubuntu с помощью VMware

Шаг 1: Откройте установленную программу vmware workstation, нажмите "Файл" в левом верхнем углу, затем "Открыть", найдите файл Ubuntu_18.04.6_VM_LinuxVMImages.COM.vmx, полученный ранее. После этого откроется новое окно диалога виртуальной машины.

![image-20230627111151531](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111151531.png)

Шаг 2: Конфигурация виртуальной машины.

На рисунке показан экран конфигурации виртуальной машины. Вы можете нажать на красное поле 2 "Редактировать параметры виртуальной машины", чтобы отрегулировать объем памяти и количество процессоров виртуальной машины. Рекомендуется выделить не менее 4 ГБ памяти и 4 процессора. После настройки нажмите "Включить эту виртуальную машину" для запуска.

![image-20230627111209633](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111209633.png)

При первом запуске появится диалог уведомления о копировании виртуальной машины. Просто нажмите "Я скопировал эту виртуальную машину" для продолжения запуска системы.

![image-20230627111219906](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111219906.png)

Дождитесь несколько секунд - система запустится автоматически. После запуска нажмите мышью на надпись "Ubuntu", чтобы перейти к диалогу входа. Введите пароль "ubuntu" для входа в систему Ubuntu.

## 2. Конфигурация зависимостей Ubuntu

Установите необходимые пакеты. Нажмите мышью в окно Ubuntu, затем одновременно нажмите клавиши Ctrl + Alt + T для быстрого открытия терминала. После открытия терминала выполните следующую команду для установки необходимых зависимостей:

```
sudo apt-get install -y  sed make binutils build-essential  gcc g++ bash patch gzip bzip2 perl  tar cpio unzip rsync file  bc wget python  cvs git mercurial rsync  subversion android-tools-mkbootimg vim  libssl-dev  android-tools-fastboot
```

![image-20230627111330777](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111330777.png)

Если вы обнаружили, что при первом запуске виртуальной машины Ubuntu не удается вставить команды, скопированные в Windows, то сначала необходимо вручную ввести и выполнить следующую команду для установки пакета, позволяющего виртуальной машине и Windows совместно использовать буфер обмена:

![image-20230627111339680](http://photos.100ask.net/allwinner-docs/v853/LinuxEnv/image-20230627111339680.png)

После завершения установки нажмите кнопку питания в верхнем правом углу для перезагрузки системы Ubuntu, или введите команду sudo reboot для перезагрузки. Теперь вы сможете копировать файлы между Windows и Ubuntu или вставлять файлы в Ubuntu.

После завершения этого шага вы готовы перейти к следующему этапу - получению исходного кода и начинаете разработку с использованием отладочной платы 100ASK_V853-Pro.

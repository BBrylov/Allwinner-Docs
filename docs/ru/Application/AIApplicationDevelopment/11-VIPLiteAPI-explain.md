# Пояснение API VIPLite

| **NPU функция**                 | **Пояснение API**                                            |
| ------------------------------- | ------------------------------------------------------------ |
| vip_init()                      | Описание:<br/>Инициализирует аппаратное обеспечение VIP и программное окружение VIPLite. Более подробно, этот API сбрасывает и инициализирует<br/>аппаратное обеспечение VIP, готовя его к приёму команд. Он также инициализирует программное окружение, такое как<br/>куча видеопамяти, управление питанием и таблицы MMU.<br/>Вызовите этот API перед вызовом любого другого API VIPLite для использования аппаратного обеспечения VIP. После<br/>завершения приложения вызовите vip_destroy().<br/>Вы можете вызывать API vip_init() несколько раз. Однако количество вызовов vip_destroy() должно быть<br/>равно количеству вызовов vip_init(). Выполняются только первый вызов vip_init() и последний вызов vip_destroy().<br/>Остальные вызовы vip_init() и vip_destroy() между ними не инициализируют и не уничтожают<br/>приложение.<br/>Синтаксис:<br/>vip_status_e vip_init(<br/>void<br/>);<br/>Параметры:<br/>Отсутствуют<br/>Возвращает:<br/>vip_status_e |
| vip_destroy()                   | Описание:<br/>Завершает работу драйвера VIPLite, освобождает ресурсы, запрошенные vip_init(), и закрывает аппаратное обеспечение VIP.<br/>Вызовите этот API после завершения приложения. После выполнения этого API вызовите vip_init()<br/>перед любым другим API VIPLite.<br/>Вы можете вызывать API vip_init() несколько раз. Однако количество вызовов vip_destroy() должно быть<br/>равно количеству вызовов vip_init(). Выполняются только первый вызов vip_init() и последний вызов vip_destroy().<br/>Остальные вызовы vip_init() и vip_destroy() между ними не инициализируют и не<br/>уничтожают приложение.<br/>Синтаксис:<br/>vip_status_e vip_destroy(<br/>void<br/>);<br/>Параметры:<br/>Отсутствуют<br/>Возвращает:<br/>vip_status_e |
| vip_create_network()            | Описание:<br/>Создаёт сеть из заданного двоичного файла. Двоичный файл — это двоичный большой объект (BLOB) данных, созданный<br/>генератором графов. Драйвер VIPLite может интерпретировать его для создания объекта сети.<br/>Синтаксис:<br/>vip_status_e vip_create_network(<br/>void<br/>*data,<br/>vip_uint32_t<br/>size_of_data,<br/>vip_enum<br/>type,<br/>vip_network<br/>*network<br/>);<br/>Параметры:<br/>ВХОД<br/>*data<br/>Указатель на двоичный файл графа.<br/>ВХОД<br/>size_of_data<br/>Размер в байтах двоичного файла графа.<br/>ВХОД<br/>type<br/>Тип сети.<br/>Поддерживаемые типы определены в<br/>перечислении vip_create_network_type_e.<br/>ВЫХОД<br/>*network<br/>Указатель для получения созданного объекта сети, если сеть<br/>создана успешно.<br/>Если создание сети не удаётся, возвращается VIP_NULL.<br/>Возвращает:<br/>vip_status_e |
| vip_destroy_network()           | Описание:<br/>Уничтожает сеть. Этот API освобождает все связанные ресурсы, выделенные для сети. После выполнения этого API для указанной сети буфер команд<br/>сети также освобождается. Однако коэффициенты скопированной сети сохраняются.<br/>Синтаксис:<br/>vip_status_e vip_destroy_network(<br/>vip_network<br/>network<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор сети, которую нужно уничтожить.<br/>Возвращает:<br/>vip_status_e |
| vip_query_network()             | Описание:<br/>Запрашивает свойства сети.<br/>Синтаксис:<br/>vip_status_e vip_query_network(<br/>vip_network<br/>network,<br/>vip_enum<br/>property,<br/>void<br/>*value<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор запрашиваемой сети.<br/>ВХОД<br/>property<br/>Свойство сети для запроса.<br/>Следующие свойства доступны для запроса:<br/>VIP_NETWORK_PROP_LAYER_COUNT<br/>VIP_NETWORK_PROP_INPUT_COUNT<br/>VIP_NETWORK_PROP_OUTPUT_COUNT<br/>VIP_NETWORK_PROP_NETWORK_NAME<br/>VIP_NETWORK_PROP_READ_REG_IRQ<br/>VIP_NETWORK_PROP_ADDRESS_INFO<br/>VIP_NETWORK_PROP_MEMORY_POOL_SIZE<br/>VIP_NETWORK_PROP_PROFILING<br/>Подробности см. в разделе 2.2.5, vip_network_property_e.<br/>ВЫХОД<br/>*value<br/>Указатель в памяти для хранения возвращаемого значения свойства.<br/>Тип данных значения зависит от запрашиваемого свойства.<br/>Возвращает: |
| vip_set_network()               | Описание:<br/>Конфигурирует сеть. Перед запуском сети необходимо вызвать vip_prepare_network().<br/>Синтаксис:<br/>vip_status_e vip_set_network(<br/>vip_network<br/>network,<br/>vip_enum<br/>property,<br/>void<br/>*value<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор конфигурируемой сети.<br/>ВХОД<br/>property<br/>Свойство сети для конфигурирования.<br/>Поддерживаемые свойства:<br/>VIP_NETWORK_PROP_CHANGE_PPU_PARAM<br/>VIP_NETWORK_PROP_SET_MEMORY_POOL<br/>VIP_NETWORK_PROP_SET_DEVICE_ID<br/>VIP_NETWORK_PROP_SET_PRIORITY<br/>Подробности см. в разделе 2.2.5, vip_network_property_e.<br/>ВХОД<br/>*value<br/>Указатель в памяти на значение свойства.<br/>Возвращает:<br/>vip_status_e |
| vip_prepare_network()           | Описание:<br/>Проверяет конфигурацию сети. Этот API выделяет ресурсы внутренней памяти для сети, разворачивает ресурсы всех операций во<br/>внутренний пул памяти, выделяет и исправляет буферы команд для ресурсов во внутреннем пуле памяти. После успешного выполнения этого API сеть<br/>считается готовой к запуску на аппаратном обеспечении VIP. Перед этим API используйте API vip_set_network() для конфигурирования сети. Если этот API вызывается<br/>несколько раз без изменения конфигурации сети, драйвер по умолчанию<br/>игнорирует вызовы API, кроме первого запроса.<br/>Синтаксис:<br/>vip_status_e vip_prepare_network(<br/>vip_network<br/>network<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор подготавливаемой сети.<br/>Возвращает:<br/>vip_status_e |
| vip_query_input()               | Описание:<br/>Запрашивает свойства входа сети.<br/>Синтаксис:<br/>vip_status_e vip_query_input(<br/>vip_network<br/>network,<br/>vip_uint32_t<br/>index,<br/>vip_enum<br/>property,<br/>void<br/>*value<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор запрашиваемой сети.<br/>ВХОД<br/>index<br/>Индекс входа сети для запроса.<br/>ВХОД<br/>property<br/>Свойство буфера входа для запроса.<br/>Следующие свойства доступны для запроса:<br/>VIP_BUFFER_PROP_QUANT_FORMAT<br/>VIP_BUFFER_PROP_NUM_OF_DIMENSION<br/>VIP_BUFFER_PROP_SIZES_OF_DIMENSION<br/>VIP_BUFFER_PROP_DATA_FORMAT<br/>VIP_BUFFER_PROP_FIXED_POINT_POS<br/>VIP_BUFFER_PROP_TF_SCALE<br/>VIP_BUFFER_PROP_TF_ZERO_POINT<br/>VIP_BUFFER_PROP_NAME<br/>VIP_BUFFER_PROP_DATA_TYPE<br/>Подробности см. в разделе 2.2.6, vip_buffer_property_e.<br/>ВЫХОД<br/>*value<br/>Указатель в памяти для хранения возвращаемого значения свойства.<br/>Возвращает:<br/>vip_status_e |
| vip_query_output()              | Описание:<br/>Запрашивает свойства выхода сети.<br/>Синтаксис:<br/>vip_status_e vip_query_output(<br/>vip_network<br/>network,<br/>vip_uint32_t<br/>index,<br/>vip_enum<br/>property,<br/>void<br/>*value<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор запрашиваемой сети.<br/>ВХОД<br/>index<br/>Индекс выхода сети для запроса.<br/>ВХОД<br/>property<br/>Свойство буфера выхода для запроса.<br/>Следующие свойства доступны для запроса:<br/>VIP_BUFFER_PROP_QUANT_FORMAT<br/>VIP_BUFFER_PROP_NUM_OF_DIMENSION<br/>VIP_BUFFER_PROP_SIZES_OF_DIMENSION<br/>VIP_BUFFER_PROP_DATA_FORMAT<br/>VIP_BUFFER_PROP_FIXED_POINT_POS<br/>VIP_BUFFER_PROP_TF_SCALE<br/>VIP_BUFFER_PROP_TF_ZERO_POINT<br/>VIP_BUFFER_PROP_NAME<br/>VIP_BUFFER_PROP_DATA_TYPE<br/>Подробности см. в разделе 2.2.6, vip_buffer_property_e.<br/>ВЫХОД<br/>*value<br/>Указатель в памяти для хранения возвращаемого значения свойства.<br/>Возвращает:<br/>vip_status_e |
| vip_set_input()                 | Описание:<br/>Подключает буфер входа к сети. При подключении буфера входа к сети драйвер VIPLite исправляет<br/>буфер команд сети для заполнения буфера входа. Вы также можете вызвать этот API для обновления буфера входа. Обновление вступит<br/>в силу со следующего сетевого выполнения. Перед подключением буфера входа к сети убедитесь, что использовали<br/>API vip_prepare_network(). Перед запуском сети с помощью vip_run_network() убедитесь, что каждый<br/>допустимый вход сети имеет подключённый буфер. В противном случае<br/>возвращается VIP_ERROR_MISSING_INPUT_OUTPUT при вызове vip_run_network() API.<br/>Синтаксис:<br/>vip_status_e vip_set_input(<br/>vip_network<br/>network,<br/>vip_uint32_t<br/>index,<br/>vip_buffer<br/>input<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор конфигурируемой сети.<br/>ВХОД<br/>index<br/>Индекс входа сети для конфигурирования.<br/>ВХОД<br/>input<br/>Непрозрачный дескриптор буфера, который будет подключён к входу сети.<br/>Возвращает:<br/>vip_status_e |
| vip_set_output()                | Описание:<br/>Подключает буфер выхода к сети. При подключении буфера выхода к сети драйвер VIPLite исправляет<br/>буфер команд сети для заполнения буфера выхода. Вы также можете вызвать этот API для обновления буфера выхода. Обновление вступит<br/>в силу со следующего сетевого выполнения. Перед подключением буфера выхода к сети убедитесь, что использовали<br/>API vip_prepare_network(). Перед запуском сети с помощью vip_run_network() убедитесь, что выход сети<br/>имеет подключённый буфер. В противном случае возвращается VIP_ERROR_MISSING_INPUT_OUTPUT<br/>при вызове API vip_run_network().<br/>Синтаксис:<br/>vip_status_e vip_set_output(<br/>vip_network<br/>network,<br/>vip_uint32_t<br/>index,<br/>vip_buffer<br/>output<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор конфигурируемой сети.<br/>ВХОД<br/>index<br/>Индекс выхода сети для конфигурирования.<br/>ВХОД<br/>output<br/>Непрозрачный дескриптор буфера, который будет подключён к выходу сети.<br/>Возвращает:<br/>vip_status_e |
| vip_run_network()               | Описание:<br/>Отправляет задачу выполнения для сети. Аппаратное обеспечение VIP выполняет задачу с наивысшим приоритетом<br/>среди отправленных задач. Вы можете<br/>вызвать этот API несколько раз.<br/>API возвращает статус выполнения после того, как аппаратное обеспечение VIP завершит выполнение. Если вам нужно<br/>немедленно вернуть статус без ожидания завершения выполнения, используйте<br/>API vip_trigger_network().<br/>Для установки приоритета сети используйте API vip_set_network(). Перед запуском сети убедитесь, что подготовили сеть с помощью vip_prepare_network(). Кроме того, через<br/>использование убедитесь, что каждый вход и выход сети имеют буфер с помощью<br/>vip_set_input() и vip_set_output(). В противном случае VIP_ERROR_MISSING_INPUT_OUTPUT<br/>возвращается при вызове vip_run_network().<br/>Для запуска нескольких сетей в одной группе используйте vip_run_group() или vip_trigger_group().<br/>Синтаксис:<br/>vip_status_e vip_run_network(<br/>vip_network<br/>network<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор запускаемой сети.<br/>Возвращает:<br/>vip_status_e |
| vip_trigger_network()           | Описание:<br/>Отправляет задачу выполнения для сети. Аппаратное обеспечение VIP выполняет задачу с наивысшим приоритетом<br/>среди отправленных задач. Вы можете<br/>вызвать этот API несколько раз.<br/>API немедленно возвращает статус выполнения без ожидания завершения аппаратным обеспечением. Для получения статуса вызовите<br/>vip_wait_network() для синхронизации. Если вам нужен статус<br/>возврата после завершения выполнения аппаратным обеспечением VIP, используйте<br/>API vip_run_network().<br/>Для установки приоритета сети используйте API vip_set_network(). Перед запуском сети убедитесь, что подготовили сеть с помощью vip_prepare_network(). Кроме того, через<br/>использование убедитесь, что каждый вход и выход сети имеют буфер с помощью vip_set_input() и vip_. В противном случае<br/>VIP_ERROR_MISSING_INPUT_OUTPUT возвращается при вызове vip_trigger_network().<br/>Для запуска нескольких сетей в одной группе используйте vip_run_group() или vip_trigger_group().<br/>Синтаксис:<br/>vip_status_e vip_trigger_network(<br/>vip_network<br/>network<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор выполняемой сети.<br/>Возвращает:<br/>vip_status_e |
| vip_wait_network()              | Описание:<br/>Ожидает завершения аппаратным обеспечением VIP выведения для указанной сети.<br/>Вызовите этот API после vip_trigger_network().<br/>Синтаксис:<br/>vip_status_e vip_wait_network(<br/>vip_network<br/>network<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор сети.<br/>Возвращает:<br/>vip_status_e |
| vip_finish_network()            | Описание:<br/>Освобождает ресурсы подготовленной сети. После вызова этого API все внутренние ресурсы памяти,<br/>выделенные для сети, освобождаются до<br/>разрушения сети. Если сеть больше не требуется, используйте<br/>API vip_destroy_network() для её уничтожения.<br/>Вызовите API vip_finish_network() для завершения подготовленной сети только когда сеть больше не используется<br/>или оставшиеся ресурсы системы ограничены для других сетей. Если сеть всё ещё требуется, не вызывайте<br/>этот API, так как подготовка сети очень затратна по времени.<br/>После успешного выполнения vip_finish_network()<br/>для подготовленной сети, повторный вызов<br/>включает тихое игнорирование, пока сеть не будет переподготовлена с помощью API vip_prepare_network().<br/>Для неподготовленной сети вызовы vip_finish_network() будут тихо игнорироваться.<br/>Важно: не вызывайте API vip_finish_network() для запущенной сети.<br/>Синтаксис:<br/>vip_status_e vip_finish_network(<br/>vip_network<br/>network<br/>);<br/>Параметры:<br/>ВХОД<br/>network<br/>Непрозрачный дескриптор завершаемой сети.<br/>Возвращает:<br/>vip_status_e |
| vip_create_buffer()             | Описание:<br/>Создаёт буфер VIP указанного размера без заполнения между строками, срезами или пакетами.<br/>Синтаксис:<br/>vip_status_e vip_create_buffer(<br/>vip_buffer_create_params_t<br/>*create_param,<br/>vip_uint32_t<br/>size_of_param,<br/>vip_buffer<br/>*buffer<br/>);<br/>Параметры:<br/>ВХОД<br/>*create_param<br/>Указатель на структуру vip_buffer_create_params_t.<br/>ВХОД<br/>size_of_param<br/>Размер структуры данных, созданной *create_param, в байтах.<br/>ВЫХОД<br/>*buffer<br/>Указатель для получения созданного объекта буфера, если буфер VIP<br/>создан успешно.<br/>Если создание буфера VIP не удаётся, возвращается VIP_NULL.<br/>Возвращает:<br/>vip_status_e<br/>Если возвращается VIP_SUCCESS, буфер VIP создан успешно.<br/>Если возвращается VIP_ERROR_<error_type>, буфер не создан. |
| vip_create_buffer_from_handle() | Описание:<br/>Создаёт буфер VIP из дескриптора и отображает физический адрес, связанный с дескриптором, на буфер.<br/>Перед использованием этого API включите VIP MMU. В противном случае API возвращает VIP_ERROR_FAILURE.<br/>Синтаксис:<br/>vip_status_e vip_create_buffer_from_handle(<br/>vip_buffer_create_params_t<br/>*create_param,<br/>vip_ptr<br/>handle_logical,<br/>vip_uint32_t<br/>handle_size,<br/>vip_buffer<br/>*buffer<br/>)<br/>Параметры:<br/>ВХОД<br/>*create_param<br/>Указатель на структуру vip_buffer_create_params_t.<br/>ВХОД<br/>handle_logical<br/>Адрес дескриптора, из которого будет создан новый буфер VIP.<br/>Для нереального времени операционная система Linux укажите логический<br/>адрес. Адрес выделяется функцией Linux malloc().<br/>Для системы реального времени укажите физический адрес.<br/>Примечание: рекомендуется выравнивание адреса на 64 байта.<br/>ВХОД<br/>handle_size<br/>Размер памяти, на которую указывает дескриптор.<br/>Примечание: рекомендуется выравнивание размера на 64 байта.<br/>ВЫХОД<br/>*buffer<br/>Указатель для получения созданного объекта буфера, если буфер VIP<br/>создан успешно.<br/>Если создание буфера VIP не удаётся, возвращается VIP_NULL.<br/>Возвращает:<br/>vip_status_e<br/>Если возвращается VIP_SUCCESS, буфер VIP создан успешно.<br/>Если возвращается VIP_ERROR_<error_type>, буфер не создан. |
| vip_destroy_buffer()            | Описание:<br/>Уничтожает буфер VIP и освобождает память, используемую буфером.<br/>Синтаксис:<br/>vip_status_e vip_destroy_buffer(<br/>vip_buffer<br/>buffer<br/>);<br/>Параметры:<br/>ВХОД<br/>buffer<br/>Непрозрачный дескриптор буфера для уничтожения.<br/>Возвращает:<br/>vip_status_e |
| vip_map_buffer()                | Описание:<br/>Создаёт указатель на указанный буфер VIP. Приложение может использовать указатель для доступа к буферу.<br/>Синтаксис:<br/>void *vip_map_buffer(<br/>vip_buffer<br/>buffer<br/>);<br/>Параметры:<br/>ВХОД<br/>buffer<br/>Непрозрачный дескриптор буфера, для которого создаётся указатель.<br/>Возвращает:<br/>Указатель на буфер, который приложения могут использовать для чтения или записи данных буфера |

| Запуск одной сети    | Процесс запуска сети подробно описан ниже:<br/>1. Вызовите vip_init() для инициализации движка VIPLite, включая программное и аппаратное обеспечение.<br/>Этот API сбрасывает аппаратное обеспечение для подготовки к использованию и инициализирует программные ресурсы. Он устанавливает видеопамять<br/>и другие ресурсы аппаратного обеспечения, такие как прерывания и регистры памяти для использования VIPLite.<br/>2. Читайте данные двоичного графа сети (NBG) из файла или памяти.<br/>3. Используйте API vip_Create_network() для создания сети.<br/>Этот API выполняет проверку здравомыслия данных NBG. Поэтому рекомендуется проверить код ошибки, возвращённый для проверки того, была ли сеть успешно создана.<br/>4. Используйте vip_Query_input() и vip_Query_output() для запроса свойств входа и выхода.<br/>Рекомендуется выполнить этот шаг, чтобы избежать ошибок, вызванных несовпадением входных или выходных свойств.<br/>5. Используйте следующие API для создания входных и выходных буферов: vip_Create_buffer(),<br/>vip_create_buffer_from_handle(), vip_create_buffer_from_physical() и<br/>vip_create_buffer_from_fd().<br/>6. (Опционально) Используйте vip_set_network() для конфигурирования свойств сети.<br/>7. Используйте API vip_Prepare_network() для подготовки буфера команд сети.<br/>Рекомендуется проверить код ошибки, возвращённый этим API. Это потому, что API может не пройти из-за ограничений ресурсов,<br/>например нехватки памяти.<br/>8. С помощью API vip_map_buffer() загрузите данные из входных и выходных буферов.<br/>9. Используйте vip_set_input() и vip_set_output() для подключения входных и выходных буферов к сети.<br/>10. Используйте API vip_Run_network() или vip_trigger_networks() для запуска сети.<br/>Если используется vip_run_network(), то при завершении выполнения аппаратным обеспечением VIP. Если используется vip_trigger_network(),<br/>она немедленно вернётся без ожидания завершения выполнения аппаратным обеспечением VIP. Если загруженность CPU велика, это<br/>оптимизирует использование CPU. В этом случае, когда CPU нуждается в результате API, вызовите vip_wait_network()<br/>для синхронизации.<br/>Примечание: можно создать несколько сетей. Однако одновременно может выполняться только одна сеть. Приложение запускает<br/>сети одну за другой в соответствии с приоритетом сети, настроенным с помощью API vip_set_network().<br/>11. Используйте vip_Flush_buffer() для очистки результатов выполнения сети из кэша CPU в выходной буфер.<br/>12. С помощью vip_map_buffer() проверьте результаты выполнения сети в выходном буфере.<br/>13. (Опционально) Повторите шаги 6-12 для многократного запуска сети.<br/>14. Вызовите vip_finish_network() для освобождения внутренней памяти, выделенной для сети.<br/>15. Вызовите vip_destroy_network() для освобождения всех остальных ресурсов, выделенных для сети.<br/>16. Вызовите vip_destroy_buffer() для освобождения памяти, выделенной входным и выходным буферам.<br/>17. Вызовите vip_destroy() для освобождения ресурсов VIPLite и выхода. |

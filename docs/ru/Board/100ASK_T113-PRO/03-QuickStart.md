# Быстрый старт использования
## Включение питания и вход в систему

###  Вход через последовательный порт

#### 1.Подключение кабеля последовательного порта

Как показано на рисунке ниже, нам нужно сначала использовать комплектный адаптер питания 12V DC для подключения к `DC разъему питания` в левой синей рамке. При этом тумблер красного разъема платы нужно переключить влево, чтобы подать питание на плату. После успешной подачи питания зеленый индикатор на основном модуле загорится. После этого мы используем TypeC кабель для подключения к `Debug интерфейсу`, показанному в синей рамке в правом нижнем углу. Один конец TypeC кабеля подключается к плате, другой конец - к USB2.0 порту компьютера.

![image-20230215173821475](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230215173821475.png)


#### 2.Установка драйвера последовательного порта

После успешного подключения кабеля type-c последовательного порта Windows автоматически установит драйвер (установка может занять около минуты).

После этого можно открыть **Диспетчер устройств** компьютера, в разделе **Порты (COM и LPT)** можно увидеть порт (**COM13**), как показано на рисунке ниже. USB чип последовательного порта на плате может быть CP210x или CH9102, их производительность одинакова. Номер COM на вашем компьютере может отличаться, запомните отображаемое число на вашем компьютере.

![image-20230215172832148](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230215172832148.png)

Если компьютер не отображает номер порта, необходимо установить драйвер вручную. Скачайте Driver Genius с официального сайта ([www.drivergenius.com](http://www.drivergenius.com)), установите, запустите, выполните проверку, и драйвер последовательного порта будет установлен автоматически.

#### 3.Запуск инструмента последовательного порта

Существует множество инструментов последовательного порта. Здесь мы рекомендуем использовать Putty или MobaXterm для входа в систему платы.

* Putty можно скачать на странице https://www.chiark.greenend.org.uk/~sgtatham/putty/.

* MobaXterm можно скачать на странице https://mobaxterm.mobatek.net/ (рекомендуется).

**Вход через последовательный порт с помощью putty**

Как показано на рисунке ниже, откройте главный интерфейс программы, следуя порядку красных рамок с номерами: **нажмите 1**, чтобы переключиться на интерфейс Serial (последовательное устройство), затем в **красной рамке 2** введите номер порта, полученный в диспетчере устройств на предыдущем шаге. В **красной рамке Speed** введите скорость передачи `115200`. После завершения базовой настройки необходимо переключить настройку управления потоком. Нажмите кнопку **Serial** в синей рамке слева программы, чтобы переключиться на интерфейс настройки последовательного устройства. После успешного переключения выберите `Flow control` в **красной рамке 3** как `None`, затем нажмите **красную рамку 4** Open, чтобы открыть соответствующее последовательное устройство платы. После этого можно нажать клавишу enter на клавиатуре и ввести ls, чтобы увидеть информацию о каталогах системы.

![image-20230215175047283](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230215175047283.png)


**Вход через последовательный порт с помощью Mobaxterm**

Откройте MobaXterm, нажмите `Session` в левом верхнем углу, во всплывающем окне выберите `Serial`. Как показано на рисунке ниже, выберите номер порта (номер порта COM13, отображаемый в диспетчере устройств), скорость передачи (Speed 115200), управление потоком (Flow Control: none), затем нажмите `OK`. Шаги показаны на рисунке ниже. **Примечание: управление потоком (Flow Control) обязательно должно быть установлено на none, иначе вы не сможете вводить данные в последовательный порт в MobaXterm**

[![Mobaxterm_serial_set_001](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/DongshanNezhaSTU/Mobaxterm_serial_set_001.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/DongshanNezhaSTU/Mobaxterm_serial_set_001.png)

После этого отобразится черное окно. В этот момент включите питание платы, и вы получите данные, отправленные последовательным портом платы, как показано на следующих шагах

![image-20230216105452371](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230216105452371.png)

#### 4.Вход в shell

После отображения информации, показанной на рисунке выше, нажмите клавишу enter на клавиатуре, чтобы войти в интерактивный интерфейс Shell системы Linux платы T113.

![image-20230216105514693](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230216105514693.png)

### Вход через adb

#### 1.Подключение OTG кабеля

Один из двух комплектных typec кабелей платы подключите непосредственно к позиции OTG, показанной в красной рамке на рисунке ниже, другой конец подключите к USB порту компьютера. Тумблер платы переключите **вправо** (в сторону OTG порта). Поскольку OTG также может питать всю плату, в этот момент можно не подключать разъем 12V DC, и система запустится автоматически.

![image-20230216105837566](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230216105837566.png)

Подождите около 10 секунд. В этот момент в диспетчере устройств вашего компьютера в разделе `Android Phone` появится устройство `Android ADB Interface`. Это устройство эмулируется платой T113 через интерфейс OTG. Через него можно входить в систему, передавать файлы и выполнять другие операции.

![image-20230216110202316](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230216110202316.png)

**Примечание: Если устройство не появляется, возможно, на компьютере по умолчанию нет соответствующего драйвера. Пожалуйста, используйте Driver Genius или другие инструменты для автоматической установки.**



#### 2.Установка ADB для Windows

Поскольку у нас есть ADB устройство, нам нужно установить инструмент adb в Windows, чтобы взаимодействовать с платой T113-PRo. Нажмите на ссылку https://gitlab.com/dongshanpi/tools/-/raw/main/ADB.7z, чтобы скачать ADB инструмент для Windows. После завершения загрузки распакуйте файл и увидите следующий каталог.

[![adb-tools-dir](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-dir.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-dir.png)

Затем скопируйте папку **platform-tools** из верхнего уровня в любой каталог. После завершения копирования запомните расположение этого каталога. Далее нам нужно добавить этот путь в переменные среды системы Windows.

[![adb-tools-dir](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-dir-001.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-dir-001.png)

Я скопировал его на диск D, мой каталог - `D:\platform-tools`. Далее мне нужно добавить его в переменные среды системы Windows, чтобы использовать команду adb в любом месте.

[![adb-tools-windows_config_001](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_001.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_001.png)

Добавление в переменные среды системы Windows [![adb-tools-windows_config_001](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_002.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_002.png)

#### 3.Открытие cmd и подключение к плате

Есть два способа открыть командную строку CMD Windows
Способ 1: Непосредственно в диалоговом окне поиска Windows10/11 введите cmd, во всплывающем окне программы нажмите `Командная строка`
Способ 2: Одновременно нажмите клавиши wind + r, введите команду cmd, нажмите подтвердить, и `Командная строка` откроется автоматически

[![adb-tools-windows_config_003](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_003.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_003.png)

Откройте командную строку, введите команду adb, и можно сразу увидеть, что наш adb успешно настроен

[![adb-tools-windows_config_004](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_004.png)](https://cdn.staticaly.com/gh/DongshanPI/Docs-Photos@master/d1s/adb-tools-windows_config_004.png)

Подключите OTG платы и подключите его к компьютеру, затем введите adb shell для автоматического входа в систему



#### 4.Вход через adb

Как показано ниже, в терминале Windows введите adb shell, чтобы автоматически войти в систему платы.

![image-20230216111155418](http://photos.100ask.net/allwinner-docs/t113-s3/basic/image-20230216111155418.png)

ADB также можно использовать для передачи файлов, например:

```
C:\System> adb push badapple.mp4 /mnt/UDISK   # Загрузить badapple.mp4 в каталог /mnt/UDISK платы
C:\System> adb pull /mnt/UDISK/badapple.mp4   # Скачать /mnt/UDISK/badapple.mp4 в текущий каталог
```

**Примечание: Этот метод в настоящее время применим только к системам, построенным с использованием Allwinner Tina-SDK.**
